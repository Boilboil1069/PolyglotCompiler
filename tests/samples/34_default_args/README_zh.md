# 34_default_args —— 命名参数与默认值

`default_args.ploy` 演示需求 2026-04-28-11 引入的命名参数 / 默认值
扩展。覆盖三类调用形态：

| 示例调用              | 等价的纯位置形式             |
| --------------------- | ---------------------------- |
| `add(10)`             | `add(10, 0)`                 |
| `add(x: 7)`           | `add(7, 0)`                  |
| `add(2, y: 5)`        | `add(2, 5)`                  |
| `scale(value: 6)`     | `scale(6, one())` → `6`      |
| `scale(4, 3)`         | `scale(4, 3)`                |

parser 与 sema 强制以下规则：

* 必需参数必须出现在带默认值的参数之前；顺序颠倒是 parse 期错误。
* 默认值表达式必须是常量可折叠的字面量 / 一元 / 二元表达式，或一次
  纯的 ploy 内函数调用（不允许跨语言 `CALL`，也不允许闭包捕获）。
  在默认值中读取另一个参数会被拒绝。
* 调用点可以自由混合位置参数和命名参数；位置参数不可出现在命名
  参数之后。每个必需（无默认值）的参数都必须由位置或名字提供。

lowering 会在每个省略对应实参的调用点物化一份默认表达式的副本，
后端因此永远看不到"短调用"。
